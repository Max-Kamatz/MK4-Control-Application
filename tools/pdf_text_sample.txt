2 Overview
The External Control Universal Protocol (EUP) is designed to allow external systems to control the NexOS
platform and receive data from it. EUP uses UDP or TCP communication, where NexOS listens on a specific
UDP / TCP port (port 34030 by default for UDP and 34030 for TCP) and responds to incoming messages.
Main principles and features:
1. String-Based Protocol: EUP is a string-based protocol, meaning messages can be read by humans
without additional preprocessing. WARNING: Each message starts with ID: and ends with end of line
symbol '\n'.
3 / 86
2. Request-Response Principle: The protocol operates on a request-response basis, meaning NexOS
does not send messages to external users on its own but only responds to incoming messages. The
NexOS has an option for notification of clients about detection results (detected objects from motion
detector, changes detector or classification). To receive notifications the client should subscribe to it.
3. Flexible Message Routing: EUP can route messages to various services based on address,
supporting different system configurations, such as:
A single main board with all services
A main board with separate payload boards for each camera
A main board with payload boards for multiple cameras
Configurations including main board, camera payload board, and non-optical payload boards,
etc.
4. Supported Message Types:
